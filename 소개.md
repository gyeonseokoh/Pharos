# Pharos

> 회의록 한 장이면 프로젝트가 굴러간다.
> 초보 팀장을 위한 AI 프로젝트 매니저, Obsidian 안에서.

---

## 한 줄 요약

Pharos는 **회의록을 던져넣으면 AI가 분석해서 Task로 쪼개고, 팀원에게 자동 배정하고, 일정까지 그려주는** Obsidian 플러그인입니다.

---

## 어떤 문제를 풀고 싶었나

학생 팀이나 사이드 프로젝트에는 PM이 따로 없습니다. 누군가 떠밀려 팀장이 되면 보통 이렇게 됩니다.

- 회의는 분명히 했는데 다음날엔 "근데 누가 뭐 하기로 했지?"
- Notion에 꼼꼼히 정리하다 지쳐서 결국 카톡으로 회귀
- GitHub Issues는 개발자만 보고, 디자이너·기획 멤버는 안 봄
- 진행도 물어보러 매번 DM 폭탄
- 개강 첫 주에 짠 일정은 2주차에 이미 무용지물

Pharos는 **"매니지하지 않아도 매니지가 되는 환경"** 을 목표로 합니다.
사용자는 회의 끝나고 회의록 한 장만 올리면 되고, 그 뒤는 AI가 처리합니다.

## Notion AI / GitHub Projects 와 뭐가 다른가

| 항목 | Notion AI | GitHub Projects | **Pharos** |
|---|---|---|---|
| 정체성 | 글쓰기 보조 도구 | 이슈 칸반 | **능동적 PM 에이전트** |
| AI 위치 | 사용자가 부르면 도와줌 | 없음 | **회의록 들어오면 자동 가동** |
| 최소 입력 단위 | 페이지·블록 | 이슈 1건 | **회의록 한 장** |
| Task 생성 방식 | 사람이 직접 | 사람이 직접 | **회의록에서 자동 추출** |
| 일정 잡기 | 수동 | 수동 | **의존 관계·기간 자동 계산** |
| 담당자 배정 | 수동 | 수동 | **팀원 기술스택 매칭으로 자동** |
| 회의 ↔ Task 연결 | 없음 | 없음 | **모든 Task에 출처 회의 링크** |
| GitHub 커밋 ↔ Task 매칭 | 없음 | 수동 링크 | **커밋 메시지 자동 인식** |
| 비개발자 친화 | 좋음 | 나쁨 | **좋음** (Obsidian 자체가 노트 도구) |
| 데이터 주권 | 클라우드 강제 | 클라우드 강제 | **로컬 우선, 동기화는 옵션** |
| 가격 | 월 $10/인 | 일정 이상 유료 | **무료 (오픈소스 지향)** |
| 진입 장벽 | 회사 SaaS 가입 | 깃허브 학습 | **Obsidian만 깔면 끝** |

### 핵심 차이를 한 줄로

- **Notion AI** → "글 잘 쓰게 도와줌"
- **GitHub Projects** → "이슈를 잘 보여줌"
- **Pharos** → **"이슈가 생기기 전 단계부터 자동화"**

다른 두 도구가 *결과물을 보기 좋게* 정리한다면, Pharos는 *결과물 자체*를 만들어 줍니다.

## 핵심 기능

### 1. 회의 → Task 자동 추출
회의록 텍스트만 던지면 AI가:
- 핵심 키워드 5개 추출
- 결정사항 자동 식별
- 언급된 기술 스택 정리
- 회의 내용을 **기능 내용 / 개발 진행도** 카테고리로 자동 분류 (다중 분류 허용)

### 2. AI 기반 로드맵 생성
누적된 회의록 + 프로젝트 마감일을 입력으로:
- MVP / 통합 / 테스트 / 배포 단계 자동 분할
- 회의록의 결정사항을 개별 Task로 분해
- 팀원 기술스택과 매칭해서 담당자 자동 배정
- 의존 관계 고려한 일정 배치 → Gantt 차트로 시각화
- PO 승인하기 전엔 미리보기 단계에서 검토 가능

### 3. 회의 주제 자동 제안 (PO-2)
"다음 회의에 뭐 다룰까?" 고민할 필요 없음.
직전 회의록 + 미해결 이슈 + 외부 자료 검색(Tavily)을 종합해서
AI가 우선순위까지 매긴 주제 후보를 제시합니다.

### 4. GitHub 커밋 자동 매칭
`feat(TASK-008): ...` 같은 컨벤션 커밋만 지키면,
매일 정해진 시각에 자동으로 GitHub에서 가져와 해당 Task에 연결.
"이 Task 진짜 했어?" 검증이 자동으로 이뤄집니다.

### 5. 회의록 자동 분류 아카이브
업로드한 회의록을 4개 탭으로 자동 정리:
- 🔧 **기능 내용** — UI·요구사항·디자인 회의
- 📈 **개발 진행도** — Task·블로커·일정 회의
- 📅 **날짜별** — 시간 역순 보기
- 📥 **기타 회의록** — 분류 안 된 회의록 + 새 회의록 추가 진입점

### 6. 대시보드 · 진행도
- Dashboard에 팀 전체 활동을 한눈에
- 내 업무 페이지에 본인 Task만 모아 보기
- 공개 진행도 페이지를 교수·팀원에게 공유

## 실제 사용 흐름

```
1. 새 프로젝트 생성
   → 주제·마감일·고정 회의 시간만 입력

2. AI가 기획 로드맵 자동 생성
   → 착수·기획·디자인·요구사항 단계 자동 분할

3. 정기/임시 회의 진행 → 회의록 작성

4. AI 자동 분석 (5단계)
   → 키워드·결정사항·기술스택·카테고리 추출
   → 회의록 관리 탭에 자동 분류

5. 충분한 회의록이 쌓이면 "개발 단계로 전환" 버튼
   → AI가 회의록 종합해서 개발 로드맵 생성
   → Task 자동 분배 + Gantt 차트

6. 팀원 GitHub 커밋 자동 수집
   → 커밋 메시지 패턴으로 Task 자동 검증

7. Dashboard에서 매일 진행도 자동 갱신
```

---

## 누구를 위한 도구인가

- **캡스톤·졸업 프로젝트 팀** — PM 경험 없는 학생들
- **사이드 프로젝트** — 친구 3~5명이 시작한 토이 서비스
- **소규모 스타트업** — Notion·Jira 도입이 부담스러운 1~10명 팀
- **연구실 프로젝트** — 논문 작성과 개발이 동시에 굴러가는 팀

## 설치

```bash
git clone https://github.com/gyeonseokoh/Pharos.git
cd Pharos
git checkout Pharos-UI/UX
npm install
npm run build
```

빌드된 `main.js` · `manifest.json` · `styles.css`를
`<옵시디언볼트>/.obsidian/plugins/pharos/` 폴더에 복사 →
옵시디언 설정에서 활성화.
